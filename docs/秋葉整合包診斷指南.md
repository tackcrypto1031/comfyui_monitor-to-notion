# 秋葉整合包 WebSocket 診斷指南

## 問題現象

- 已連接 ComfyUI
- 只收到 `crystools.monitor` 消息（每秒 2 次）
- 沒有收到 `execution_start`, `executing`, `progress`, `status` 消息
- 算圖時狀態沒有變化

## 可能原因

### 1. 秋葉整合包使用不同的 WebSocket 消息格式

秋葉整合包可能修改了 ComfyUI 的 WebSocket 消息系統。

### 2. 需要啟用 WebSocket 狀態推送

某些整合包需要手動啟用狀態推送功能。

### 3. 需要通過 HTTP API 查詢狀態

整合包可能只支持 HTTP API 輪詢，不支持 WebSocket 推送。

## 診斷步驟

### 步驟 1: 檢查 ComfyUI 控制台

在遠端 ComfyUI 的控制台中，應該可以看到類似以下的日誌：

```
WebSocket message sent: execution_start
WebSocket message sent: executing
WebSocket message sent: progress
```

如果沒有這些日誌，表示整合包沒有發送標準消息。

### 步驟 2: 檢查秋葉整合包設定

打開秋葉整合包，檢查是否有以下設定：

1. **WebSocket 設定**
   - 是否有「啟用 WebSocket 狀態推送」選項
   - 是否有「發送執行狀態」選項

2. **插件設定**
   - 是否有 ComfyUI 監控相關插件
   - 是否有自訂 WebSocket 插件

### 步驟 3: 測試 HTTP API

如果 WebSocket 不工作，我們可以改用 HTTP API 輪詢：

```bash
# 獲取隊列狀態
curl http://192.168.25.108:8188/prompt

# 獲取系統狀態
curl http://192.168.25.108:8188/system_stats
```

### 步驟 4: 檢查 ComfyUI 版本

秋葉整合包可能使用較舊或較新版本的 ComfyUI，消息格式可能不同。

## 解決方案

### 方案 A: 通過 HTTP API 輪詢狀態

如果 WebSocket 不推送狀態，我們可以定期通過 HTTP API 查詢：

```typescript
// 每 2 秒輪詢一次
setInterval(async () => {
  const response = await fetch(`http://${ip}:${port}/prompt`);
  const data = await response.json();
  // 處理隊列狀態
}, 2000);
```

### 方案 B: 使用 ComfyUI 標準版本

如果秋葉整合包不兼容，可以考慮使用標準 ComfyUI。

### 方案 C: 檢查整合包文檔

查看秋葉整合包的文檔或 GitHub，確認 WebSocket 消息格式。
