### 狀態顏色說明

| 顏色 | 狀態 | 說明 |
|------|------|------|
| 🟢 綠色 | 閒置中 (Idle) | 無任務執行 |
| 🟡 黃色 | 運行中 (Running) | 有任務在隊列中 |
| 🔵 藍色 | 算圖中 (Generating) | 正在執行生成任務 |
| ⚫ 灰色 | 未連接 (Disconnected) | 未連接或離線 |
| 🔴 紅色 | 錯誤 (Error) | 連接錯誤 |

## 添加機器

### 步驟

1. **點擊「+ 添加機器」按鈕**
   - 位於右上角的藍色按鈕

2. **填寫機器資訊**
   ```
   名稱：ComfyUI-01          (必填，1-50 字元)
   IP 地址：192.168.1.100    (必填，IPv4 格式)
   埠號：8188                (必填，1-65535)
   ```

3. **驗證輸入**
   - 名稱不能為空
   - IP 必須是有效的 IPv4 格式
   - 埠號必須在 1-65535 範圍內

4. **點擊「添加」**
   - 成功後自動連接到該機器
   - 卡片出現在列表中

### 編輯機器

1. 目前版本暫不支援直接編輯
2. 請先刪除再重新添加

### 刪除機器

1. 點擊機器卡片右上角的 🗑️ 圖示
2. 確認刪除對話框
3. 機器將從列表中移除

## 連接管理

### 單一機器連接/斷開

1. 點擊機器卡片上的 🔌 圖示
2. **已連接時**: 點擊斷開連接
3. **未連接時**: 點擊建立連接

### 全部連接

1. 點擊頁首的 **「全部連接」** 按鈕
2. 所有未連接的機器將嘗試連接
3. 連接狀態即時更新

### 全部斷開

1. 點擊頁首的 **「全部斷開」** 按鈕
2. 所有連接將被斷開
3. 機器狀態變更為「未連接」

### 連接狀態

| 狀態 | 顯示 | 說明 |
|------|------|------|
| 已連接 | 綠色文字 | WebSocket 連接成功 |
| 連接中 | 黃色文字 + 旋轉圖示 | 正在建立連接 |
| 未連接 | 灰色文字 | 未嘗試連接 |
| 錯誤 | 紅色文字 | 連接失敗，顯示錯誤訊息 |

### 自動重連

- 斷線後自動嘗試重連
- 重連策略：指數退避 (1s → 2s → 4s → 8s → 16s → 30s)
- 最多重連 10 次
- 30% 抖動避免同時重連

## 狀態監控

### 狀態識別邏輯

**閒置中 (Idle) 🟢**
- 隊列為空 (`queue_remaining = 0`)
- 無正在執行的節點

**運行中 (Running) 🟡**
- 隊列有任務 (`queue_remaining > 0`)
- 或有節點正在執行但無進度更新

**算圖中 (Generating) 🔵**
- 有節點正在執行
- 且有進度更新 (`progress > 0`)

### 狀態更新頻率

- WebSocket 即時推送
- 狀態變化 <100ms 反映到 UI
- 節流機制：500ms 最小間隔

### 狀態列資訊

狀態列顯示即時統計：

```
機器總數：5 | 已連接：5 | 閒置：2 | 運行：1 | 算圖：2
```

- **機器總數**: 已添加的機器總數
- **已連接**: 當前連接成功的機器數
- **閒置**: 處於閒置狀態的機器數
- **運行**: 處於運行狀態的機器數
- **算圖**: 處於算圖狀態的機器數

### 最後更新時間

每個機器卡片顯示最後更新時間：
- `< 60 秒`: 顯示「X 秒前」
- `>= 60 秒`: 顯示「X 分鐘前」

## Notion 整合

### 配置步驟

1. **開啟設定**
   - 點擊右上角「⚙️ 設定」按鈕

2. **選擇 Notion 整合頁籤**

3. **填寫配置**
   ```
   Notion Integration Token: secret_xxxxx...
   資料庫 ID: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```

4. **測試連接**
   - 點擊「測試連接」按鈕
   - 成功顯示「✓ 連接成功」
   - 失敗顯示錯誤訊息

5. **儲存配置**
   - 點擊「儲存配置」
   - Token 將加密儲存

### 自動記錄

配置完成後：
- 每次狀態變化自動建立 Notion 頁面
- 記錄內容包含：
  - 機器名稱
  - 狀態 (閒置/運行/算圖)
  - IP 位置
  - 時間戳記
  - 連接狀態
  - 前一個狀態

### 查看記錄

在 Notion 資料庫中：
- 每筆記錄為一個頁面
- 可按時間、狀態、機器篩選
- 可匯出為 CSV/Excel

詳細設置請參考 [Notion 設置指南](notion-setup.md)

## 提示與技巧

### 最佳實踐

1. **機器命名**: 使用有意義的名稱 (如：位置_用途)
   - ✅ `Studio_Main`, `Home_Backup`
   - ❌ `Machine1`, `Test`

2. **定期檢查**: 定期查看狀態列確保所有機器正常連接

3. **Notion 備份**: 定期匯出 Notion 記錄作為備份

4. **資源管理**: 建議最多監控 20 台機器以保持最佳效能

### 故障排除

遇到問題時：
1. 檢查機器 IP 和埠號是否正確
2. 確認 ComfyUI 正在運行
3. 檢查防火牆設定
4. 查看 [故障排除指南](troubleshooting.md)
